## ✨ Key Capabilities

### 🔬 End-to-End Machine Learning Pipelines
Every project covers the complete ML lifecycle: raw data ingestion → parsing → feature engineering → EDA → modelling → evaluation → visualisation → AI-assisted business synthesis → export-ready deliverables.

### 🤖 LLM-Assisted Analysis
Gemini 2.5 Flash and OpenAI GPT-4o provide automated dataset profiling, hypothesis generation, and executive narrative synthesis — compressing hours of analytical work into seconds.

### 📊 Automated Exploratory Data Analysis
Four-pass AutoViz framework provides automated chart generation (KDE distributions, scatter plots, heatmaps, violin plots, pair plots, bar charts, box plots) across multiple analytical angles without manual chart coding.

### 🔍 Explainable AI (SHAP & LIME)
Production-grade dual-explainability strategy: SHAP TreeExplainer for global feature attributions and regulatory documentation; LIME LimeTabularExplainer for patient-level, threshold-based clinical communication.

### 🏆 Automated Model Selection
H2O AutoML trains and benchmarks GBMs, XGBoost, Random Forests, Deep Learning, and Stacked Ensembles in a single call — surfacing the best-performing model with full leaderboard transparency.

### 📈 Interactive Visualisation Dashboards
70+ publication-quality plots across three projects, covering sentiment distributions, model confidence histograms, temporal heatmaps, SHAP beeswarms, LIME bar charts, waterfall decompositions, ROC/PR curves, PDP grids, and residual diagnostics.

### 📄 Academic-Grade Reporting
Each project is paired with a comprehensive academic PDF report covering mathematical formulations, results tables, figure captions, business interpretation, limitations, and deployment recommendations.

## 📡 Project 1 — Customer Sentiment Intelligence on Twitter

> *AI-powered NLP pipeline for automated sentiment classification and executive business intelligence extraction from customer support tweets.*

[![Open in Colab](https://img.shields.io/badge/Open%20Notebook-Google%20Colab-F9AB00?style=for-the-badge&logo=googlecolab&logoColor=white)](https://drive.google.com/file/d/1L9GF9A3ZPBAJlP-OHRDJaanD0edQIQ5o/view?usp=sharing)
&nbsp;
[![Academic Report](https://img.shields.io/badge/Academic%20Report-PDF-DC143C?style=for-the-badge&logo=adobeacrobatreader&logoColor=white)](customer_sentiment_twitter/docs/customer_sentiment_twitter.pdf)

### Problem Statement
At scale, manual Twitter sentiment review is operationally infeasible. Without automated classification, negative sentiment signals accumulate unreported until they appear in quarterly NPS surveys — too late for intervention. This project closes the gap from raw tweet stream to board-ready customer intelligence in a reproducible, auditable pipeline.

### Pipeline Architecture
```
Raw CSV → Parsing → Feature Engineering (17 features) → AutoViz EDA (4 passes)
       → RoBERTa Inference (batch=64) → Visualisation → Gemini/GPT-4o Synthesis → CSV Export
```

### Feature Engineering (17 Features, 5 Groups)
| Group | Features |
|---|---|
| Text Structure | `text_length`, `word_count`, `hashtag_count`, `punct_count` |
| Engagement Signals | `has_mention`, `has_url`, `uppercase_ratio` |
| Linguistic Quality | `stopword_ratio`, `lexical_richness`, `avg_word_length` |
| VADER Sentiment | `sent_pos`, `sent_neu`, `sent_neg`, `sent_compound` |
| Temporal | `hour`, `day_of_week`, `is_weekend` |

### AutoViz EDA Framework
| Pass | Target | Analytical Goal | Chart Types |
|---|---|---|---|
| **Pass 1** | `inbound_int` | Customer vs. agent feature separation | KDE, bar, scatter, heatmap, box, pair |
| **Pass 2** | `sent_compound` | VADER sentiment score drivers (LOWESS) | Scatter+LOWESS, violin, dist, heatmap |
| **Pass 3** | `word_count` | Tweet verbosity predictors | Scatter+LOWESS, box, dist, bar, pair |
| **Pass 4** | *(none)* | Unsupervised distributional exploration | Dist, heatmap, pair, violin, bar |

### Model: cardiffnlp/twitter-roberta-base-sentiment-latest
- Pre-trained on **124 million tweets**, fine-tuned on TweetEval benchmark
- **~72–75% macro F1** on TweetEval sentiment task
- 3-class output: Positive · Neutral · Negative
- Batch inference: `batch_size=64`, max 128 tokens (covers 99.7% of tweets)

### Key Results
| Metric | Value |
|---|---|
| Tweets classified | 10,000 |
| Negative sentiment | 50–65% |
| Avg. model confidence | >0.80 across all classes |
| Neutral confidence spread | 0.50–0.70 (semantic ambiguity) |
| Temporal pattern | Pre-06:00 UTC shows elevated negative ratio |

## 🫀 Project 2 — Heart Disease Predictive Analytics with Explainable AI

> *Dual-model classification pipeline with SHAP and LIME explainability for transparent, regulation-ready clinical decision support.*

[![Academic Report](https://img.shields.io/badge/Academic%20Report-PDF-DC143C?style=for-the-badge&logo=adobeacrobatreader&logoColor=white)](heart_disease_shap_lime/reports/docs/heart_disease_shap_lime.pdf)

### Problem Statement
In clinical AI, accuracy alone is insufficient. A model assigning high risk without explaining *which signals drove that decision* cannot be trusted, audited under the EU AI Act (Article 13), or safely deployed. This project pairs ensemble classifiers with SHAP and LIME to produce a transparent, patient-level explanation alongside every prediction.

### Dataset
**UCI Heart Disease Dataset** — 303 patients across 4 clinical sites (Cleveland, Budapest, Long Beach VA, Zurich), 13 clinical features + binary heart disease label.

### Feature Engineering
Three age-mediated interaction terms are constructed beyond the 13 raw features:

```
age_x_oldpeak   = age × oldpeak         # Compounded ST depression risk in older patients
thalch_div_age  = thalch / age          # Age-adjusted maximum heart rate
chol_div_age    = chol / age            # Age-adjusted cholesterol
```

### Model Performance

| Metric | Random Forest | XGBoost |
|---|---|---|
| **AUC-ROC** | **0.932** | 0.916 |
| **Recall** | **0.902** | 0.863 |
| **F1-Score** | **0.880** | 0.859 |
| Precision | 0.860 | 0.854 |
| Specificity | 0.817 | 0.817 |
| MCC | **0.725** | 0.681 |
| Brier Score | **0.108** | 0.112 |
| TP / FP / TN / FN | 92/15/67/10 | 88/15/67/14 |

> **Clinical recommendation:** Random Forest for screening (highest Recall = fewest missed disease cases). XGBoost for regulatory documentation (stronger SHAP magnitudes, sharper nonlinear thresholds).

### SHAP Analysis
- **Global importance:** Both models consistently rank `cp_asymptomatic`, `exang`, `age_x_oldpeak`, and `thalch_div_age` as dominant risk signals — cross-model consensus validates genuine disease relationships
- **Beeswarm plots:** Patient-level SHAP distributions reveal directional feature effects across the full test set
- **Dependence plots:** 2×4 grid (RF + XGB) visualises age-mediated amplification of ST depression risk
- **Waterfall plots:** Per-patient decomposition from base value to final prediction for Patients 0, 4, and 6

### LIME Analysis
- `LimeTabularExplainer`: `num_features=12`, `num_samples=500`
- Threshold-based rule format (e.g., `cp_asymptomatic > 0.5`) for clinical communication
- LIME magnitudes are 2–5× larger than SHAP (local linear amplification near decision boundary)
- Stochastic: seed must be fixed for reproducible production explanations

### SHAP vs. LIME Comparison
| Aspect | SHAP | LIME |
|---|---|---|
| Theoretical basis | Shapley values (game theory) | Local linear approximation |
| Scope | Global + Local | Local only |
| Stability | High | Moderate (stochastic) |
| Representation | Continuous contributions | Threshold-based rules |
| Clinical use | Regulatory / audit | Patient communication |

## 🏡 Project 3 — Advanced House Sale Price Prediction with AutoML

> *Featuretools Deep Feature Synthesis + H2O AutoML pipeline for automated model selection, ensemble learning, and SHAP-proxy interpretability on the Ames housing dataset.*

[![Academic Report](https://img.shields.io/badge/Academic%20Report-PDF-DC143C?style=for-the-badge&logo=adobeacrobatreader&logoColor=white)](predicting_house_sale_prices/reports/docs/predicting_house_sale_prices.pdf)

### Pipeline Architecture
```
Raw Data → SweetViz EDA → Preprocessing → Featuretools DFS
         → Feature Selection → H2O AutoML → Leaderboard Evaluation
         → Champion Model Analysis → SHAP Proxy → PDP → Predictions Export
```

### Automated Feature Engineering (Featuretools DFS)
- Deep Feature Synthesis generates hundreds of candidate features from raw primitives (aggregations, transforms, cross-entity features)
- Primitive analysis ranks features by predictive signal
- Correlation filtering removes redundant features
- Threshold evaluation exports: `dfs_features_selected.csv`, `dfs_features_discarded.csv`, `dfs_threshold_evaluation.csv`

### H2O AutoML
H2O AutoML trains and benchmarks the following algorithm families in a single call:
- Gradient Boosting Machines (GBM)
- XGBoost
- Random Forest / Distributed RF
- Deep Learning
- Generalised Linear Models (GLM)
- **Stacked Ensembles (BestOfFamily + AllModels)**

**Champion model:** `StackedEnsemble_BestOfFamily_1_AutoML_2_20260406_182447`

Full leaderboard exported to `reports/metrics/h2o_leaderboard.csv` for transparency and reproducibility.

### Model Interpretability
- **H2O Variable Importance:** Global feature ranking across the champion ensemble
- **Variable Importance Aggregated:** Consensus importance across top leaderboard models
- **SHAP Proxy:** SHAP-style attribution approximation for the H2O ensemble
- **Partial Dependence Plots (PDP):** Marginal effect of top features on predicted price
- **Residual Analysis:** Diagnostic plots (residuals vs. fitted, Q-Q plot) confirming model assumptions

## 💼 Business Impact

### Project 1 — Customer Sentiment Intelligence

**Problem solved:** Manual Twitter sentiment review at 4,200 tweets/weekend is operationally impossible. This pipeline classifies 10,000 tweets in under 10 minutes with >0.80 average confidence.

**Quantifiable value:**
- A reduction from 60% to 50% negative sentiment represents a **17% relative improvement** in brand sentiment baseline — directly trackable via the pipeline
- Hourly analysis identifies the **00:00–06:00 UTC window** as a high-distress staffing gap, enabling empirically grounded SLA differentiation
- AI synthesis (Gemini + GPT-4o) compresses 2–3 hours of executive report writing to under 3 minutes

### Project 2 — Heart Disease Predictive Analytics

**Problem solved:** Clinical AI without explanation cannot satisfy EU AI Act Article 13 or FDA SaMD guidance. This pipeline attaches a ranked, signed feature explanation to every prediction.

**Quantifiable value:**
- Random Forest identifies **4 additional true disease cases per 61 patients** compared to XGBoost (Recall 0.902 vs. 0.863) — each missed case represents a patient who receives no intervention
- SHAP waterfall plots enable clinicians to **see, challenge, and override** model decisions with full situational awareness
- LIME threshold-based output enables direct patient communication: *"Your normal thalassaemia test result is the strongest factor reducing your predicted risk"*

### Project 3 — House Sale Price Prediction

**Problem solved:** Manual feature engineering and model selection are time-consuming and incomplete. Featuretools + H2O AutoML systematically explore the feature and model space, surfacing solutions a human analyst would likely miss.

**Quantifiable value:**
- Automated feature engineering generates and evaluates hundreds of candidate features in minutes
- H2O leaderboard provides full model transparency — every algorithm family benchmarked against the same holdout set
- SHAP proxy + PDP grid delivers interpretable feature effects on price, enabling appraisal reasoning and risk communication
